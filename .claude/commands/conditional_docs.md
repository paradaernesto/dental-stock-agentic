# Conditional Documentation Guide

Based on the upcoming task, read the relevant documentation:

## For Planning Tasks
Read:
- specs/ folder structure
- Any existing related specs

## For Implementation Tasks  
Read:
- Relevant spec file
- lib/ structure
- app/ structure

## For Testing Tasks
Read:
- tests/ structure
- vitest.config.ts

## For Review Tasks
Read:
- The spec being reviewed
- Related implementation files

## For Documentation Tasks
Read:
- README.md
- Any feature specs
- app_docs/ if exists
