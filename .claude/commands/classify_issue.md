# GitHub Issue Command Selection

Based on the `Github Issue` below, follow the `Instructions` to select the appropriate command.

## Instructions

- Based on the details in the `Github Issue`, select the appropriate command
- IMPORTANT: Respond exclusively with '/' followed by the command
- Use the command mapping to help you decide

## Command Mapping

- Respond with `/chore` if the issue is maintenance, refactoring, config, docs
- Respond with `/bug` if the issue is a bug fix
- Respond with `/feature` if the issue is a new feature
- Respond with `/patch` if the issue requests a quick patch
- Respond with `0` if none of the above
